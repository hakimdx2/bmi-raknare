## 3. 🟡 `/bmi-barn/` — CTR 1.8% (40 clics / 2 208 impr)

### Diagnostic
- **Titre actuel** : `BMI Barn – ISO-BMI Kalkylator & Räknare (Juli 2026)`
- **Longueur** : 59 caractères ✅
- **Sur 1 ligne** : ❌ **MULTI-LIGNE** — le `</title>` est sur une ligne séparée
- **Meta description** : ✅ Présente, 1 ligne
- **Position** : 10.7 (page 1 bas / page 2)

**Problème principal** : Titre sur 2 lignes dans le HTML source. Google parse un texte avec saut de ligne → CTR dégradé. Malgré 2 208 impressions et une position correcte (10.7), seuls 40 clics (1.8%).

**Cause racine** : Le template `segment.twig` utilise `{% block title %}{{ data.title }}{% endblock %}`. Le `data.title` est généré dynamiquement mais le rendu Twig introduit un saut de ligne entre le texte et le `</title>`. Le cache Twig sur le serveur Hostinger est probablement **stale** — le template source dans le repo est correct (1 ligne), mais le serveur sert une version compilée obsolète.

### ✅ Fix en 2 étapes

**Étape 1 — Template `templates/pages/segment.twig`** (ligne 41) : Ajouter le contrôle de whitespace Twig
```diff
-{% block title %}{{ data.title }}{% endblock %}
+{%- block title -%}{{ data.title }}{%- endblock -%}
```

## 4. 🔴 CANONICAL CASSÉ — TOUTES les pages

### Problème

Le `<link rel="canonical">` dans `base.twig` est rendu sur **3 lignes** avec un **TAB** dans le href :

```html
<link
rel="canonical" href="	https://bmi-raknare.se/
">
```

Le TAB (entre `href="` et `https://`) rend l'URL canonique invalide pour Google. C'est pire que pas de canonical du tout — Google voit une URL cassée.

### Cause racine

Les blocs `{% block canonical %}` dans les templates enfants (home.twig:14-16, ozempic.twig:14-15, segment.twig, etc.) sont sur plusieurs lignes avec indentation. L'indentation (TAB) est injectée dans le href.

### ✅ Fix en 2 étapes

**Étape 1 — `templates/base.twig` ligne 17** : déjà sur 1 ligne dans le repo ✅

```html
<link rel="canonical" href="{% block canonical %}{{ base_url }}/{% endblock %}">
```

**Étape 2 — TOUS les templates enfants** : mettre le bloc canonical sur 1 ligne

```diff
# home.twig
-{% block canonical %}
-	{{ base_url }}/
-{% endblock %}
+{% block canonical %}{{ base_url }}/{% endblock %}

# ozempic.twig
-{% block canonical %}
-	{{ base_url }}/bmi-ozempic-krav/
-{% endblock %}
+{% block canonical %}{{ base_url }}/bmi-ozempic-krav/{% endblock %}

# segment.twig, aldre.twig, gravid.twig, etc. — même pattern
```
